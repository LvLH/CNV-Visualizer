igv

Browser

Initializes the basic values for the browser itself, as well as setting up mouse events and other handlers.

options
Set up initially
trackContainerDiv
Div that is meant to contain the tracks themselves

disableZoomWidget()

Turns off the widget that allows zooming in and out

enableZoomWidget(zoomHandlers)

Turns on the widget that allows zooming in and out

zoomHandlers
All pieces that need to be capable of zooming in/out

toggletCursorGuide(genomicStateList)

Enable/Disable cursor guides

genomicStateList
All pieces with loci to show a line through the cursor

toggleCenterGuide(genomicStateList)

Enable/Disable guide through the center of the track

genomicStateList
All pieces with loci to be split

getFormat(name)

Returns a format as long as it was supplied from the options list.

name
the format that is requested
return ->
the format

loadTracksWithConfigList(configList)

Load several tracks that are passed in as a list of configurations for each individual track. Calls igv.Browser.loadTrack(config)

configList
An array of configurations for each respective track
return ->
all of the tracks that have been loaded

loadTrack(config)

Loads a specific track with a given configuration, requires a case added to the config types

config
the configuration passed in for the track
return ->
the track that's been loaded

addTrack(track)

Add a track to the igv viewer, and ensure it fits properly into the containing div element. Calls igv.Browser.reorderTracks()

track
the track that has been completely built to be added

reorderTracks()

Sort the tracks and ensure that they are in said order in the containing div

removeTrack(track)

Remove a selected track, and take it out of the container div.

track
track to remove; should be in the

findTracks(property, value)

Search for tracks that contain a particular value

property
property to search for
value
expected value for the property
return ->
the tracks with the requested value

reduceTrackOrder(trackView)

Move a track up the view to be higher up (lower order?) Calls igv.Browser.reorderTracks()

trackView
the track to be moved

increaseTrackOrder(trackView)

Move a track down the view to be lower (higher order?) Calls igv.Browser.reorderTracks()

trackView
the track to be moved

setTrackHeight(newHeight)

Set a new uniform height for the tracks

newHeight
pixel? height of the tracks

resize()

Calls a resize on everything renderable in the browser

repaint()

Calls a repaint on everything renderable in the browser

repaintWithLocusIndex(locusIndex)

Calls a repaint on Karyo, Ideo, and Viewports with a locus index

locusIndex
the index that needs to be repainted

update()

Updates the Locus Search, then updates the Genomic State, then renderables and finally each of the individual tracks

updateWithLocusIndex(locusIndex)

Update the genomicState, then a specific locus of the genomicState, then Ideo, Karyo, and Viewports. Update the center guide as well

locusIndex
the index of the genome

loadInProgress()

Check if any tracks are still trying to load and return that

return ->
boolean whether loading or not

updateLocusSearchWithGenomicState()

Updates the current locus being searched for data, and fire the event

syntheticViewportContainerBBox()

Object that describes the position and size of the div containing the genome viewport.

return ->
rect with { position, width, height }

syntheticViewportContainerWidth()

Get the width of the viewport containing the tracks

return ->
number with width

viewportContainerWidth()

Get the visible width of a track (should be the same for all of them)

return ->
number width of the viewport

minimumBasesExtent()

return ->
40...?

goto(chrName, start, end)

Updates the genomicState to go straight to a specified location given by the chromosome, start base pair and end base pair. Ends by running igv.Browser.update().

chrName
name of the chromosome
start
start base pair
end
end base pair

zoomIn()

Zoom into the tracks, only if all tracks have already finished loading

zoomOut()

Zoom out of the tracks, only if all tracks have finished loading

selectMultiLocusPanelWithGenomicState(genomicState)

Create a panel of locuses that match the given Genomic State. Calls igv.Browser.multiLocusPanelLayoutWithTruthFunction(fn).

genomicState
state variable to try matching

closeMultiLocusPanelWithGenomicState(genomicState)

Close the panel of viewports that match the genomicState. Do this by only leaving all of the other genomicStates open. Calls igv.Browser.multiLocusPanelLayoutWithTruthFunction(fn).

genomicState
state variable to try matching

multiLocusPanelLayoutWithTruthFunction(filterFunction)

Filter genomicStates possible using the given function. Build the relevant viewport. Calls igv.Browser.emptyViewportContainers(trackContainer). Then calls igv.Browser.buildViewportsWithGenomicStateList(gsList)

filterFunction
function used to filter the genomicStates

emptyViewportContainers(trackContainer)

Remove everything from a specific track container.

trackContainer
container to empty

buildViewportsWithGenomicStateList(genomicStateList)

For each of the track views, update the relevant genomicStates.

genomicStateList
Array of genomicState objects that define what should be

placed in the viewport

parseSearchInput(string)

Calls igv.Browser.getGenomicStateList(loci, viewportContainerWidth, continuation) to fill the track containers with anything from the passed in string of loci.

string
Loci used in the search

getGenomicStateList(loci, viewportContainerWidth, continuation)

Takes the loci and maps it to a list of relevant genomicStates.

loci
array of locus strings (e.g. 'chr1:1-100', 'egfr')
viewportContainerWidth
width of the viewport in pixels
continuation
callback to receive the genomicStates list.

object to set as the scope for the event handler

Ensure that the extent (start, end) of the chromosome is valid in the browser's igv.Browser.minimumBasesExtent(). Fix the extent to be valid once the function finishes.

chromosome
the chromosome we're checking
extent
start, end range to check
